# Requisitos Funcionais
#### A _Tabela 1_ a seguir contém os **Requisitos Funcionais (RF)** elicitados.

| ID   |             Requisitos              | Prioridade |
| :--: | :---------------------------------: | :--------: |
| RF01 | O sistema deve permitir que um novo usuário se cadastre fornecendo as informações necessárias (ex: nome, senha/PIN).    | Alta       |
| RF02 | O sistema deve permitir que um usuário cadastrado realize login para acessar suas informações. | Alta |
| RF03 | O sistema deve permitir que o usuário adicione um novo medicamento à sua lista, informando nome, dosagem, forma e instruções.  | Alta |
| RF04 | O sistema deve exibir ao usuário a lista de todos os seus medicamentos cadastrados.  | Alta |
| RF07 | O sistema deve permitir que o usuário crie um agendamento para um medicamento, definindo horários e frequência.  | Alta |
| RF09 | O sistema deve disparar um alerta visual (pop-up) no horário programado para cada lembrete de medicamento.  | Alta |
| RF10 | O sistema deve permitir que, no alerta, o usuário confirme que tomou o medicamento (ação "Já tomei").  | Alta |
| RF08 | O sistema deve apresentar ao usuário uma tela principal com a visualização de todos os lembretes agendados para o dia atual.  | Média |
| RF11 | O sistema deve atualizar a tela principal para indicar visualmente que um lembrete foi confirmado pelo usuário. | Média |
| RF05 | O sistema deve permitir que o usuário edite as informações de um medicamento previamente cadastrado. | Média |
| RF06 | O sistema deve permitir que o usuário remova um medicamento de sua lista.  | Média |
| RF11 | O sistema deve permitir que, no alerta, o usuário adie a notificação por um curto período de tempo (ação "Adiar 5 min").   | Média |
